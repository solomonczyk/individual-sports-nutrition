# Детальный план разработки приложения (мультиязычная версия)

## Условия

* Команда: 1 человек (ты) + ИИ
* График: 4 часа в день
* Подход: интенсивный, без лишних шагов
* Цель: MVP за минимально возможное время
* Языки: сербский, венгерский, румынский, английский, русский, украинский

## Общая структура этапов

1. **Подготовка и проектирование** — 12 дней (учет мультиязычности)
2. **Backend (API + база) MVP** — 20 дней
3. **Frontend (мобильное приложение) MVP** — 26 дней
4. **AI-модули и RAG** — 16 дней
5. **Интеграции, тестирование, оптимизация** — 12 дней
6. **Запуск и аналитика** — 6 дней

---

# 1. Подготовка и проектирование (12 дней)

### День 1–4 — ТЗ + структура данных

* Финализация функциональных блоков
* ER-диаграмма
* Структура API
* Описание AI-модулей
* Учёт мультиязычности: ключи локализации, словари, тексты UI

### День 5–8 — Архитектура

* Модульная схема backend
* Архитектура мобильного клиента
* Слой интеграций
* План DevOps-процессов
* Подготовка структуры для перевода на 6 языков

### День 9–12 — UI/UX прототип

* Прототип всех экранов с возможностью переключения языка
* UX-потоки: онбординг → ввод данных → подбор питания → рекомендации
* Готовность для переноса в разработку

---

# 2. Backend MVP (20 дней)

### День 13–16 — База данных

* PostgreSQL
* Таблицы: пользователи, продукты, добавки, рецепты, противопоказания, переводы
* Миграции

### День 17–20 — Авторизация и профиль

* Регистрация (email/Google/Apple)
* Профиль здоровья с выбором языка
* API-валидаторы

### День 21–26 — Нутриционная логика

* Расчёт калорий
* Расчёт БЖУ по целям
* Локализация под сербскую кухню
* Поддержка всех языков UI и текстовых сообщений

### День 27–32 — Подбор спортивного питания

* Алгоритм оценки потребностей
* Правила противопоказаний
* Система рекомендаций на всех языках

---

# 3. Frontend (мобильное приложение) MVP (26 дней)

### День 33–38 — Онбординг и ввод данных

* Flow из 8+ экранов
* Формы, валидации, сохранение профиля с учетом языка

### День 39–44 — Главный экран и рекомендации

* Карточки дневных рационов на выбранном языке
* Карточки спортивного питания с переводом

### День 45–50 — Трекинг

* Графики прогресса на выбранном языке
* История потребления

### День 51–56 — Настройки

* Профиль
* Цели
* Уведомления и смена языка

---

# 4. AI-модули (16 дней)

### День 57–61 — Модель рекомендаций

* Векторная база
* RAG
* Алгоритм персонализации с учётом языка

### День 62–66 — Проверка противопоказаний

* Логика сопоставления
* Объяснения на выбранном языке

### День 67–72 — Генератор рационов

* Автоматизация меню под сербскую кухню и локальные продукты
* Учет тренировок
* Перевод меню на все поддерживаемые языки

---

# 5. Интеграции, тестирование, оптимизация (12 дней)

### День 73–78

* Интеграции с трекерами (Health Connect / HealthKit)
* Логи и мониторинг
* Оптимизация API и мультиязычности

### День 79–84

* Полное тестирование всех языков
* UX-правки
* Финальный проход по логике питания

---

# 6. Запуск (6 дней)

### День 85–90

* Подготовка описаний и скриншотов на всех языках
* Публикация в Google Play / App Store
* Пилотный запуск
* Аналитика первых пользователей

---

# Итог

**Общее время: 90 дней (≈ 3 месяца)**

4 часа в день → ≈ 360 часов разработки.

Проект полностью реализуем при твоём участии + поддержке ИИ на всех этапах, с поддержкой 6 языков.
